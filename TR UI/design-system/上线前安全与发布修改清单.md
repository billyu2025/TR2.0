# 上线前安全与发布修改清单

本文档汇总 **必须修复的高风险项**、**强烈建议项** 与 **建议发布顺序**，供工程与发布负责人逐项落实与签字。

**数据库（当前约定）：** 生产与日常主路径为 **PostgreSQL**（`DB_BACKEND=postgres`，连接串见环境变量 `POSTGRES_DSN`）。SQLite 仅作兼容/本地备选；备份、监控、迁移与容量规划以 PostgreSQL 为准。

**关联代码/配置：** `backend/tr_fill_in_api.py`、`backend/db_adapter.py`、`.env`、Windows 服务与调度入口、反向代理（Nginx/IIS）

---

## 一、上线前必须完成（高风险 / 阻塞发布）

| # | 问题 | 修改要求 | 验收标准 |
|---|------|----------|----------|
| M1 | 明文密码入库与经接口返回 | 去掉 `password_plaintext` 的存储与任何 API 返回；`list_users()` / `create_user()` / `update_user()` 等不得返回 `current_password` 或等价明文 | DB 仅存 hash + salt；网络与日志中无明文密码；管理员仅能通过「重置密码」等安全流程处理 |
| M2 | CORS 全开放 | 将 `origins: "*"` 改为 **正式环境域名白名单**；若必须带 cookie/凭证，**禁止** `*` + `supports_credentials=True` 的组合 | 仅允许配置的域名跨域；非白名单来源浏览器请求被拒绝 |
| M3 | 生产 DEBUG 与错误泄露 | 生产 `.env` 中 **`DEBUG=False`**（或等价关闭）；`DEBUG_MODE` 关闭时 **不向客户端返回 traceback/内部堆栈** | 异常时返回统一、无内部路径与源码信息的错误结构；预发/生产复测 |
| M4 | 会话 token 来源过宽 | **禁止**从 URL query（如 `?token=`）读取会话；移除或忽略 query token 分支 | 仅接受 `Authorization: Bearer`（及若保留则仅限 header 的 `X-Auth-Token`，按你们最终规范二选一或并存但**绝不走 URL**） |
| M5 | 限流未生效 | 生产环境 **安装并启用 `flask-limiter`**，确保日志中 **不再出现** `flask-limiter not installed, rate limiting disabled` | 登录、下载、批量/重接口有独立限流策略；压测下触发 429 行为符合预期 |
| M6 | 调度器 / 自动更新未接入生产入口 | 将自动更新、定时任务接到 **真实生产启动链路**；服务重启后 **自动恢复**，不依赖手工启动脚本 | 重启机器或服务后任务按时运行；失败可观测（日志/指标） |
| M7 | 外网传输与暴露面 | **HTTPS**；**Waitress 等应用端口不直接对公网**，置于 Nginx/IIS 反向代理后 | 公网仅 443；证书有效；后端仅内网或本机回环监听（按架构） |
| M8 | 管理员接口暴露面过大 | 对管理类接口加强保护：**IP 白名单**、**二次验证**、或 **仅内网/VPN 可访问**（至少选一种可落地的组合） | 外网普通用户无法直接调用高危管理接口；有变更与审计记录 |

## 二、强烈建议补齐（上线后风险仍高，建议同迭代完成）

| # | 项 | 修改/工作内容 | 验收标准 |
|---|----|---------------|----------|
| S1 | 权限与越权 | 针对 **普通用户 / manager / admin** 做一轮 **越权测试**：数据范围、他人订单、系统更新类接口 | 测试用例与结果留档；已知问题为 0 或已修复并有豁免说明 |
| S2 | 备份与恢复 | **PostgreSQL**（业务库、用户/会话表）、日志、生成文件目录等做 **备份策略 + 恢复演练** | RPO/RTO 有书面说明；恢复演练成功记录 |
| S3 | 监控与告警 | 至少：**进程存活**、**端口**、**磁盘**、**数据库文件/目录**、**任务失败率**、**登录失败率** | 告警能触达负责人；误报在可接受范围 |
| S4 | 日志脱敏 | 日志中不出现 **token**、**密码**、客户敏感字段、**完整服务器路径**（按你们规范屏蔽或截断） | 抽检生产或预发日志无敏感明文 |
| S5 | 并发与压力 | 重点：**批量下载**、**PDF 生成**、**文件索引更新** 等 | 有压测报告或基线数据；瓶颈与限流策略一致 |
| S6 | 客户交付物 | **域名**、**正式证书**、**用户手册**、**管理员手册**、**故障恢复手册**、**升级/回滚流程** | 交付清单勾选完成 |

## 三、建议发布顺序

1. **先修安全项（阻塞）**  
   M1 密码与返回 → M2 CORS → M3 DEBUG/错误 → M4 URL token → M5 限流 → M7 HTTPS/反代 → M8 管理端保护  

2. **再修稳定性项**  
   M6 调度器/自动更新接入生产入口；结合 S3 监控告警  

3. **验证与质量**  
   S1 权限测试、S5 压力测试、S4 日志脱敏抽检  

4. **交付与演练**  
   S2 备份恢复、S6 文档与交付物  

5. **最后对客户开放外网**  
   安全项全部验收后再切流量或开放账号
