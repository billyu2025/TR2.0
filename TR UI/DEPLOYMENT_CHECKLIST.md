# TR系统部署前检查清单

## 🔒 安全性检查

### 1. 认证与授权
- [ ] **修改默认管理员密码**
  - 确保所有默认账号（admin, superadmin）的密码已修改为强密码
  - 密码应包含大小写字母、数字和特殊字符，长度至少12位

- [ ] **检查用户权限**
  - 确认所有客户账号的权限设置正确（user/manager/admin）
  - 禁用不需要的测试账号
  - 检查每个账号的 Job_No 权限范围

- [ ] **会话管理**
  - 确认 `SESSION_TTL_HOURS` 设置合理（建议24小时）
  - 检查会话过期后的处理逻辑

### 2. 网络安全
- [ ] **启用HTTPS（强烈建议）**
  - 当前使用HTTP，生产环境应配置SSL/TLS证书
  - 配置Nginx支持HTTPS（443端口）
  - 强制HTTP重定向到HTTPS

- [ ] **CORS配置**
  - 当前配置：`origins: "*"`（允许所有来源）
  - **建议修改**：限制为特定域名
  ```python
  CORS(app, resources={r"/*": {"origins": ["https://yourdomain.com", "https://www.yourdomain.com"]}}, supports_credentials=True)
  ```

- [ ] **防火墙配置**
  - 确认Windows防火墙规则正确配置
  - 只开放必要的端口（8000用于Nginx，5000用于Flask内部）
  - 5000端口不应直接暴露给外网

- [ ] **路由器/网关配置**
  - 确认端口转发规则正确（外网端口 → 内网IP:8000）
  - 考虑使用非标准端口增加安全性

### 3. 数据安全
- [ ] **数据库备份**
  - 设置定期自动备份数据库
  - 备份文件应存储在安全位置
  - 测试备份恢复流程

- [ ] **敏感信息保护**
  - 确认 `.env` 文件不在版本控制中
  - 检查代码中是否有硬编码的密码或密钥
  - 确保数据库文件权限设置正确

- [ ] **SQL注入防护**
  - 确认所有数据库查询使用参数化查询（已实现）
  - 检查是否有直接拼接SQL的地方

### 4. 输入验证
- [ ] **API输入验证**
  - 确认所有API端点都有输入验证
  - 检查文件上传大小限制
  - 验证文件类型限制

## 🚀 性能优化

### 1. 服务器配置
- [ ] **关闭调试模式**
  - 确认 `DEBUG_MODE = False` 或 `DEBUG=false` 在环境变量中
  - 移除或注释掉所有 `print()` 调试语句（或改为日志）

- [ ] **数据库优化**
  - 确认数据库索引已创建
  - 检查慢查询日志
  - 考虑定期清理旧数据

- [ ] **Nginx优化**
  - 启用gzip压缩
  - 配置静态文件缓存
  - 调整worker_processes和worker_connections

### 2. 资源限制
- [ ] **文件大小限制**
  - 确认PDF生成和下载的文件大小限制合理
  - 设置Nginx的 `client_max_body_size`

- [ ] **并发限制**
  - 考虑添加请求速率限制（rate limiting）
  - 监控服务器资源使用情况

4. **定期备份数据库**，防止数据丢失
