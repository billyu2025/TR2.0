# Redis 存儲位置說明

## Redis 存儲機制

### 核心概念

Redis 是一個**內存數據庫**，數據主要存儲在**內存（RAM）**中，但也可以持久化到**磁盤**。

## 存儲位置

### 1. 主要存儲：內存（RAM）

**位置：**
- Redis 數據主要存儲在**服務器內存**中
- 不是存儲在硬盤文件

**特點：**
- ✅ **速度極快**：內存讀寫速度遠快於硬盤
- ✅ **高性能**：可以達到每秒數十萬次操作
- ⚠️ **易失性**：服務器重啟或斷電，內存數據會丟失（除非啟用持久化）

**類比：**
- 就像電腦的 RAM（內存）
- 關機後數據會消失（除非保存到硬盤）

### 2. 持久化存儲：磁盤文件（可選）

**位置：**
- 如果啟用持久化，數據會保存到**磁盤文件**
- 文件位置取決於 Redis 配置

**Windows 系統：**
```
C:\Program Files\Redis\dump.rdb
或
C:\Redis\dump.rdb
```

**Linux 系統：**
```
/var/lib/redis/dump.rdb
或
/usr/local/redis/dump.rdb
```

**macOS 系統：**
```
/usr/local/var/db/redis/dump.rdb
```

## Redis 持久化機制

### 方式1：RDB（Redis Database）

**工作原理：**
- 定期將內存中的數據**快照**保存到磁盤文件（.rdb 文件）
- 類似於"拍照"保存當前狀態

**特點：**
- ✅ 文件小，恢復快
- ✅ 適合備份
- ⚠️ 可能丟失最後一次快照之後的數據

**配置示例：**
```
# 每 900 秒（15分鐘）至少 1 個 key 改變，就保存
save 900 1

# 每 300 秒（5分鐘）至少 10 個 key 改變，就保存
save 300 10

# 每 60 秒至少 10000 個 key 改變，就保存
save 60 10000
```

**文件位置：**
- 默認文件名：`dump.rdb`
- 位置：Redis 配置文件中的 `dir` 設置

### 方式2：AOF（Append Only File）

**工作原理：**
- 記錄每個**寫入操作**到日誌文件（.aof 文件）
- 類似於"記錄所有操作日誌"

**特點：**
- ✅ 數據更安全（最多丟失 1 秒數據）
- ✅ 可以重放操作恢復數據
- ⚠️ 文件較大，恢復較慢

**配置示例：**
```
appendonly yes
appendfsync everysec  # 每秒同步一次
```

**文件位置：**
- 默認文件名：`appendonly.aof`
- 位置：Redis 配置文件中的 `dir` 設置

### 方式3：無持久化

**工作原理：**
- 數據僅存儲在內存中
- 不保存到磁盤

**特點：**
- ✅ 性能最好
- ❌ 服務器重啟後數據全部丟失
- ⚠️ 僅適合臨時數據

## Redis 數據存儲流程

### 正常運行時

```
用戶請求 → Redis 讀取/寫入
  ↓
數據存儲在內存中
  ↓
（如果啟用持久化）
  ↓
定期或實時保存到磁盤文件
```

### 服務器重啟時

```
服務器啟動 → Redis 啟動
  ↓
（如果啟用持久化）
  ↓
從磁盤文件（.rdb 或 .aof）恢復數據
  ↓
加載到內存中
  ↓
Redis 恢復到重啟前的狀態
```

## 實際存儲位置示例

### Windows 系統

**Redis 安裝位置：**
```
C:\Program Files\Redis\
```

**數據文件位置（如果啟用持久化）：**
```
C:\Program Files\Redis\dump.rdb        # RDB 快照
C:\Program Files\Redis\appendonly.aof # AOF 日誌
```

**配置文件位置：**
```
C:\Program Files\Redis\redis.conf
```

### Linux 系統

**Redis 安裝位置：**
```
/usr/local/redis/
或
/etc/redis/
```

**數據文件位置：**
```
/var/lib/redis/dump.rdb
/var/lib/redis/appendonly.aof
```

**配置文件位置：**
```
/etc/redis/redis.conf
```

## Redis vs 其他存儲方式

### Redis（內存數據庫）

| 特性 | 說明 |
|------|------|
| **主要存儲** | 內存（RAM） |
| **持久化** | 可選（RDB/AOF） |
| **速度** | 極快（內存讀寫） |
| **容量** | 受內存限制 |
| **數據丟失風險** | 無持久化時：重啟丟失 |

### SQLite（文件數據庫）

| 特性 | 說明 |
|------|------|
| **主要存儲** | 磁盤文件（.db） |
| **持久化** | 自動（文件即持久化） |
| **速度** | 較慢（磁盤讀寫） |
| **容量** | 受磁盤空間限制 |
| **數據丟失風險** | 低（文件即數據） |

### 文件系統緩存

| 特性 | 說明 |
|------|------|
| **主要存儲** | 磁盤文件 |
| **持久化** | 自動 |
| **速度** | 慢（文件 I/O） |
| **容量** | 受磁盤空間限制 |
| **數據丟失風險** | 低 |

## Redis 內存管理

### 內存限制

**配置：**
```
maxmemory 256mb  # 最大使用 256MB 內存
```

**超過限制時：**
- Redis 會根據策略刪除數據
- 策略包括：LRU（最近最少使用）、LFU（最少使用頻率）等

### 內存使用情況

**查看命令：**
```bash
redis-cli INFO memory
```

**輸出示例：**
```
used_memory:1048576        # 已使用內存（字節）
used_memory_human:1.00M    # 已使用內存（人類可讀）
maxmemory:268435456        # 最大內存限制
maxmemory_human:256.00M    # 最大內存限制（人類可讀）
```

## 數據文件管理

### RDB 文件（dump.rdb）

**特點：**
- 二進制格式
- 文件大小取決於數據量
- 可以手動備份和恢復

**備份：**
```bash
# 手動觸發保存
redis-cli SAVE

# 或異步保存（不阻塞）
redis-cli BGSAVE
```

**恢復：**
- 將 `dump.rdb` 文件放到 Redis 數據目錄
- 重啟 Redis，自動加載

### AOF 文件（appendonly.aof）

**特點：**
- 文本格式（可讀）
- 文件可能較大
- 記錄所有寫入操作

**優化：**
```bash
# 重寫 AOF 文件（壓縮）
redis-cli BGREWRITEAOF
```

## 實際應用場景

### 場景1：僅內存存儲（無持久化）

**適用：**
- 臨時緩存數據
- 會話數據（Session）
- 不需要持久化的數據

**特點：**
- 性能最好
- 重啟後數據丟失

### 場景2：內存 + RDB 持久化

**適用：**
- 需要持久化但可以容忍少量數據丟失
- 數據恢復速度要求高

**特點：**
- 性能好
- 可能丟失最後一次快照後的數據
- 恢復快

### 場景3：內存 + AOF 持久化

**適用：**
- 數據安全性要求高
- 不能容忍數據丟失

**特點：**
- 數據安全
- 可能丟失最多 1 秒數據
- 恢復較慢

### 場景4：內存 + RDB + AOF（雙持久化）

**適用：**
- 數據安全性要求極高
- 需要快速恢復

**特點：**
- 最安全
- 恢復快（優先使用 RDB）
- 配置複雜

## 總結

### Redis 存儲位置

1. **主要存儲：內存（RAM）**
   - 數據主要存在服務器內存中
   - 速度極快，性能最好

2. **持久化存儲：磁盤文件（可選）**
   - RDB 文件：`dump.rdb`（快照）
   - AOF 文件：`appendonly.aof`（操作日誌）
   - 位置：Redis 配置的 `dir` 目錄

### 關鍵要點

- ✅ **數據主要存在內存中**（速度快）
- ✅ **可以持久化到磁盤**（可選，防止數據丟失）
- ✅ **文件位置取決於 Redis 配置**
- ⚠️ **無持久化時，重啟後數據會丟失**

### 建議

- **生產環境**：啟用持久化（RDB 或 AOF）
- **開發環境**：可以不用持久化（性能更好）
- **重要數據**：使用 AOF 或雙持久化
- **臨時數據**：可以不用持久化

**簡單來說：** Redis 數據主要存儲在**內存**中，如果啟用持久化，會保存到**磁盤文件**（.rdb 或 .aof），文件位置取決於 Redis 的配置。
