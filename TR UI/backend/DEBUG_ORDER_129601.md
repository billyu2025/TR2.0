# Order 129601 调试指南

## 问题
下载 order 129601 时，只下载了 `SS79270_HL2310_18_OCT_2025`，没有下载 `CMMSC2502935000-002` 和 `SSSTE2500551S`。

## 调试步骤

### 1. 查看后端日志

下载 order 129601 时，查看后端日志中的以下信息：

#### IAT 文件查找日志
```
[IAT下载] Order 129601: 开始查找 IAT Formal 文件
[IAT下载] 关键词列表: [...]
[IAT下载] IAT Formal 找到 X 个文件: [...]
```

**检查点**：
- 是否找到了所有 3 个文件？
- 如果只找到了 1 个文件，说明 `find_files_by_keywords` 有问题

#### 文件匹配日志
```
[IAT下载] 警告: 文件 XXX 无法匹配到 stockist_cert
[文件匹配] 发现 X 个无法匹配到 stockist_cert 的文件
[文件匹配] 未匹配文件: XXX (路径: ...)
```

**检查点**：
- 哪些文件无法匹配到 stockist_cert？
- 这些文件的路径是什么？

#### 文件分配日志
```
[文件匹配] 按文件夹分组: X 个文件夹
[文件匹配] 文件夹 XXX: X 个文件
[修复] 将文件夹 XXX 中的 X 个未匹配文件分配给 XXX
[文件分配] 最终文件分配情况:
[文件分配] XXX: X 个文件
```

**检查点**：
- 未匹配的文件是否被正确分配到文件夹？
- 是否找到了同一文件夹中已匹配的文件？
- 未匹配的文件是否被分配给 stockist_cert？

#### ZIP 创建日志
```
[ZIP创建] 处理 stockist_cert: XXX, X 个文件
[ZIP创建] 已添加文件: XXX -> XXX/XXX
[下载] 实际添加到 ZIP 的文件数: X
```

**检查点**：
- 哪些文件被添加到 ZIP？
- 实际添加到 ZIP 的文件数是多少？

### 2. 可能的问题和解决方案

#### 问题 1: 文件没有被找到
**症状**：日志显示只找到了 1 个文件

**可能原因**：
- 文件不在 HL2310 文件夹中
- 文件在其他位置（如 IAT Prelim）
- 文件夹名称不匹配

**解决方案**：
- 检查文件实际位置
- 检查文件夹名称是否包含关键词

#### 问题 2: 文件被找到但无法匹配
**症状**：日志显示找到了 3 个文件，但只有 1 个匹配成功

**可能原因**：
- 文件名不包含 stockist_cert 或 rm_dn_no
- 文件路径不包含关键词

**解决方案**：
- 检查未匹配文件的路径
- 确认文件是否在同一个子文件夹中

#### 问题 3: 文件被分配但未添加到 ZIP
**症状**：日志显示文件被分配，但 ZIP 中没有

**可能原因**：
- ZIP 创建时出错
- 文件路径问题

**解决方案**：
- 检查 ZIP 创建日志中的错误信息
- 检查文件路径是否正确

### 3. 手动验证

1. **检查文件位置**：
   - 确认 `CMMSC2502935000-002` 和 `SSSTE2500551S` 是否在 `IAT Formal/HL2310` 文件夹中
   - 如果不在，记录实际位置

2. **检查数据库**：
   - 查询 order 129601 的 stockist_cert 和 rm_dn_no
   - 确认这些值是否在文件名或路径中

3. **检查文件夹结构**：
   - 确认 HL2310 文件夹的完整路径
   - 确认文件是否在子目录中

如果按照上述步骤仍然无法解决问题，请提供：
