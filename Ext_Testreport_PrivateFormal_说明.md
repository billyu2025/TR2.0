# Ext_Testreport_PrivateFormal.py 提取原理说明

## 概述

`Ext_Testreport_PrivateFormal.py` 是一个文件提取脚本，用于从网络共享目录中提取 Private Formal 类型的测试报告 PDF 文件，并复制到本地目标目录。

## 提取原理

### 1. 源目录（Source Directory）

**网络共享路径**：
```
\\192.168.32.212\TVSC-Internal\Dp-Supply Chain\Inventory\07 Stockist and Mill cert Packages to Customer\02 - With Private Report
```

**说明**：
- 这是一个网络共享目录（UNC 路径）
- 位于内部服务器 `192.168.32.212`
- 路径中包含 "With Private Report"，表示这些是包含 Private Report 的证书包

### 2. 目标目录（Target Directory）

**本地路径**：
```
D:\Stockist&Test Report\Private Formal
```

**说明**：
- 本地存储 Private Formal 类型测试报告的目录
- 如果目录不存在，脚本会自动创建

### 3. 提取内容

#### 文件类型
- **只提取 PDF 文件**（`.pdf` 扩展名）

#### 文件命名模式
- **纯 DN_No 格式**：文件名只包含字母和数字，以 `.pdf` 结尾
- **正则表达式模式**：`^[A-Z0-9]+\.pdf$`（不区分大小写）

#### 匹配示例
✅ **匹配的文件名**：
- `SS78156.pdf`
- `C0146.pdf`
- `SS76288.pdf`
- `ZZ4306.pdf`

❌ **不匹配的文件名**：
- `SS78156_Private_Formal.pdf`（包含下划线和文本）
- `Stockist + MIll Cert_C0483.pdf`（包含空格和特殊字符）
- `Physical, chemical & geometry test report of SS79438.pdf`（包含逗号和空格）

### 4. 提取流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 扫描源目录（递归搜索所有子目录）                          │
│     \\192.168.32.212\...\02 - With Private Report          │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 过滤 PDF 文件                                            │
│     只处理 *.pdf 文件                                        │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 匹配命名模式                                             │
│     使用正则表达式 ^[A-Z0-9]+\.pdf$ 检查文件名              │
│     只保留纯 DN_No 格式的文件                                │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 去重检查（通过数据库索引）                                │

└─────────────────────────────────────────────────────────────┘
```

### 5. 去重机制

脚本使用数据库索引（`file_index_cache` 表）进行智能去重：

**去重逻辑**：
1. 从文件名中提取标识符（如 `SS78156`、`C0146`）
2. 查询数据库，检查 Private Formal 文件夹中是否已存在包含相同标识符的文件
3. 如果找到对应文件（即使文件名不同），跳过复制

**示例**：
- 源文件：`C0146.pdf`
- 目标目录中已有：`C0146_Private_Formal.pdf`
- 结果：跳过复制（识别为同一文件）

### 6. 文件处理策略

| 情况 | 处理方式 |
|------|---------|
| 目标文件已存在（同名） | 跳过复制 |
| 找到对应文件（不同名但同一内容） | 跳过复制 |
| 目标文件不存在 | 复制文件 |
| 复制失败 | 记录错误，继续处理下一个文件 |

**重要**：
- ✅ **不删除源文件**：只复制，不移动
- ✅ **保留文件元数据**：使用 `shutil.copy2()` 保留修改时间等
- ✅ **自动创建目标目录**：如果目标目录不存在，会自动创建

## 代码关键部分

### 1. 文件匹配模式

```python
FILE_PATTERN = re.compile(
    r'^[A-Z0-9]+\.pdf$',
    re.IGNORECASE
)
```

### 2. 递归搜索

```python
# 递归搜索所有PDF文件
for pdf_file in source_path.rglob("*.pdf"):
    file_name = pdf_file.name
    # 检查文件名是否匹配模式（纯DN_No命名）
    if FILE_PATTERN.match(file_name):
        matching_files.append((str(pdf_file), file_name))
```

### 3. 去重检查

```python
# 检查 Private Formal 文件夹中是否已存在对应文件
existing_file = index_query.check_file_exists_in_private_formal(
    file_name, base_folder
)
```

### 4. 文件复制

```python
# 复制文件（保留元数据）
shutil.copy2(source_path, target_file)
```

## 使用场景

### 适用场景
- ✅ 从网络共享目录批量提取 Private Formal 测试报告
- ✅ 自动整理和归档测试报告文件
- ✅ 避免重复复制已存在的文件

### 不适用场景
- ❌ 提取其他格式的文件（如 Word、Excel）
- ❌ 提取非纯 DN_No 格式的文件
- ❌ 需要删除源文件的场景

## 注意事项

1. **网络连接**：需要能够访问网络共享目录 `\\192.168.32.212\...`
2. **数据库依赖**：去重功能需要数据库索引，如果数据库不可用，会跳过去重检查
3. **文件权限**：需要读取源目录和写入目标目录的权限
4. **文件命名**：只提取纯 DN_No 格式的文件，其他格式的文件会被忽略
